## 任务二：ADB Shell 中文输入重连修复

### 根因分析

```
中文输入 → IME compositionstart → composing=true
→ xterm.js 处理合成文本 → 触发 onData
→ compositionend → setTimeout(50ms) 重置 composing
→ 竞态：若 onData 在 50ms 内触发，composing=true → 输入被丢弃
→ ADB shell 无输入 → 进程超时/管道断开
→ 用户看到 "Device Disconnected"，无法重连
```

### 实施计划

#### Phase 1: 修复 IME 竞态（立即修复）

**文件: `src/pages/Devices.tsx` (lines 159-163)**

将 `setTimeout(() => { composing.current = false }, 50)` 替换为：

```typescript
textarea.addEventListener('compositionend', () => {
  // rAF 在浏览器处理完所有待处理的输入事件后触发
  // 比 setTimeout 更可靠，因为它与浏览器事件循环同步
  requestAnimationFrame(() => {
    composing.current = false
  })
})
```

**更稳健的备选方案**（如果 rAF 仍有边界情况）：

```typescript
textarea.addEventListener('compositionend', () => {
  const onInput = () => {
    composing.current = false
    textarea.removeEventListener('input', onInput)
  }
  textarea.addEventListener('input', onInput, { once: true })
  setTimeout(() => {
    textarea.removeEventListener('input', onInput)
    composing.current = false
  }, 200) // 安全超时
})
```

#### Phase 2: 添加自动重连（Electron 模式）

**文件: `electron/main.cjs`**
- 新增 `adb:shell:reconnect` IPC handler
- 逻辑：kill 旧进程 → spawn 新的 `adb -s <serial> shell` → 返回新 shellId
- `proc.on('close')` 传递退出码给渲染进程

**文件: `electron/preload.cjs`**
- 暴露 `adbShellReconnect(serial, oldId)` 方法

**文件: `src/types/index.ts`**
- `ElectronAPI` 接口添加 `adbShellReconnect` 类型

**文件: `src/pages/Devices.tsx`**
- `ShellPanel` 添加 `disconnected` 状态
- 显示 "设备已断开" + "重连" 按钮
- 重连逻辑：调用 `adbShellReconnect` → 更新 shellId → 重新注册监听器
- `Devices` 组件管理重连状态（shellId 需要可变，通过 ref 或提升状态）

#### Phase 3: Web 模式重连增强
