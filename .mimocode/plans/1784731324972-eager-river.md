# 服务器连接 & 部署修复方案

## 现状分析

| 问题 | 根因 | 影响 |
|------|------|------|
| SSH 无法连接 | Windows 私钥文件权限过宽（BUILTIN\Administrators 有 Full Control） | 无法远程操作服务器 |
| 前端 404 | 静态文件路径修复未在容器内生效（未 `--build`） | 浏览器显示 "Cannot GET /" |
| OAuth 回调超时 | docker-compose 未暴露 3000 端口，但 FEISHU_REDIRECT_URI 指向 :3000 | 飞书回调无法抵达服务器 |
| Docker 健壮性 | Dockerfile 缺少 `ENV NODE_ENV=production` | 直接 run 镜像时前端不加载 |

---

## 第一步：修复 SSH 连接（本地执行）

Windows 终端执行：
```powershell
# 修复私钥权限（仅当前用户可读）
icacls $env:USERPROFILE\.ssh\id_ed25519 /reset
icacls $env:USERPROFILE\.ssh\id_ed25519 /inheritance:r /grant:r "$env:USERNAME:(R)"

# 创建 SSH config（确保使用正确密钥）
@"
Host server
    HostName 122.51.90.193
    User ubuntu
    IdentityFile ~/.ssh/id_ed25519
    IdentitiesOnly yes
    StrictHostKeyChecking accept-new
"@ | Out-File -Encoding ASCII $env:USERPROFILE\.ssh\config

# 测试连接
ssh server hostname
```

---

## 第二步：服务器端修复（SSH 连接后执行）

### 2.1 暴露 3000 端口（OAuth 回调）

```bash
cd /home/ubuntu/TsToolsPlatform
sed -i '/app:/,/restart:/ s|restart: unless-stopped|ports:\n      - "3000:3000"\n    restart: unless-stopped|' docker-compose.yml
```

### 2.2 修复前端静态文件路径

```bash
sed -i "s|path.join(__dirname, '..', 'dist')|path.join(__dirname, '..')|" server/index.ts
sed -i "s|if (fs.existsSync(staticPath))|if (fs.existsSync(path.join(staticPath, 'index.html')))|" server/index.ts
```

### 2.3 Dockerfile 添加 NODE_ENV

```bash
sed -i '/^WORKDIR \/app$/a ENV NODE_ENV=production' Dockerfile
```

### 2.4 更新 .env 回调地址

.env 中 FEISHU_REDIRECT_URI 已指向 :3000，不变。

### 2.5 重新构建并启动

```bash
docker compose up -d --build
```

## 第三步：验证

```bash
# 1. 健康检查
curl http://122.51.90.193:3000/health
# 期望: {"status":"ok","timestamp":...}

# 2. 前端页面
curl -s http://122.51.90.193/ | grep -o '<title>[^<]*'
# 期望: TsCameraTools（而非 Error）

# 3. 数据库
docker compose exec postgres psql -U postgres -d tscameratools -c "\dt"
# 期望: users, sessions, device_history, _migrations

# 4. 容器状态
docker compose ps
# 期望: 三个服务全部 Up
```

2. 点击飞书登录 → 跳转授权
3. 授权后 → 回调到 122.51.90.193:3000/auth/feishu/callback → 重定向到主页
