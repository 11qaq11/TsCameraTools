# 数据库 + Docker 部署 + 飞书用户持久化方案

## 一、技术选型

| 组件 | 选择 | 理由 |
|------|------|------|
| 数据库 | **PostgreSQL** (pg 库) | 支持多实例部署、高并发、生产级可靠性 |
| ORM | **不用 ORM** | 直接写 SQL，符合 ponytail 原则，项目数据模型简单 |
| 数据库迁移 | 手动 SQL 脚本 | 项目规模小，不需要 Prisma/Drizzle 等重型工具 |
| 容器化 | **Docker + docker-compose** | 编排 app + PostgreSQL + Nginx 三个服务 |
| 反向代理 | **Nginx** | 前端静态文件托管 + API/WS 反向代理 |
| Linux 二进制 | **手动放入 bin/** | ttyd-linux、platform-tools-linux 随代码提交 |

## 二、数据库设计

### 2.1 连接方式
- 使用 `pg` 库的 `Pool` 连接池
- 连接字符串从环境变量 `DATABASE_URL` 读取
- Docker 部署时 PostgreSQL 作为独立服务

### 2.2 表结构

#### users 表 — 飞书用户信息
```sql
CREATE TABLE IF NOT EXISTS users (
  id            SERIAL PRIMARY KEY,
  feishu_id     VARCHAR(64) NOT NULL UNIQUE,
  name          VARCHAR(128) NOT NULL,
  email         VARCHAR(256) DEFAULT '',
  avatar        VARCHAR(512) DEFAULT '',
  tenant_key    VARCHAR(64) DEFAULT '',
  created_at    TIMESTAMP DEFAULT NOW(),
  last_login_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_users_feishu_id ON users(feishu_id);
CREATE INDEX idx_users_tenant_key ON users(tenant_key);
```

#### sessions 表 — 登录会话
```sql
CREATE TABLE IF NOT EXISTS sessions (
  id            VARCHAR(64) PRIMARY KEY,
  user_id       INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  created_at    TIMESTAMP DEFAULT NOW(),
  expires_at    TIMESTAMP NOT NULL
);
CREATE INDEX idx_sessions_user_id ON sessions(user_id);
CREATE INDEX idx_sessions_expires_at ON sessions(expires_at);
```

#### device_history 表 — 设备连接历史
```sql
CREATE TABLE IF NOT EXISTS device_history (
  id            SERIAL PRIMARY KEY,
  user_id       INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  device_serial VARCHAR(128) NOT NULL,
  device_model  VARCHAR(128) DEFAULT '',
  connected_at  TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_device_history_user_id ON device_history(user_id);
```

## 三、后端改造

### 3.1 新增文件
```
server/
├── db/
│   ├── index.ts           # PostgreSQL 连接池（单例）
│   ├── migrations.ts      # 迁移执行器
│   └── migrations/
│       └── 001_init.sql   # 初始表结构
├── middleware/
│   └── auth.ts            # 认证中间件（session 验证）
├── routes/
│   └── user.ts            # 用户信息和设备历史查询 API
```

### 3.2 修改文件清单

#### server/config.ts
新增配置项：
```typescript
database: {
  url: process.env.DATABASE_URL || 'postgres://postgres:postgres@localhost:5432/tscameratools'
},
session: {
  expiryHours: parseInt(process.env.SESSION_EXPIRY_HOURS || '24')
}
```

#### server/index.ts
- 启动时初始化数据库连接池
- 执行数据库迁移
- 生产模式下添加 `express.static` 托管前端

#### server/routes/auth.ts
改造认证流程：
- `GET /feishu/callback`：登录成功后 UPSERT users 表 + 创建 session 记录
- `GET /me`：从 sessions 表验证 token 有效性，JOIN users 返回完整信息
- `POST /logout`：删除 sessions 记录
- Token 改为 UUID（替代 Base64 编码）

#### server/routes/adb.ts
- 设备连接时 INSERT device_history 记录

#### src/utils/auth.ts
- 前端无需大改，token 格式从 Base64 变为 UUID，存储逻辑不变

### 3.3 认证流程对比

**当前（不安全）：**
```
飞书OAuth → Base64(user) 作为 token → localStorage
GET /me → Base64 解码 → 返回（无验证、无过期、可伪造）
```

**改造后（安全）：**
```
飞书OAuth → UPSERT users → INSERT sessions(UUID, expires_at) → localStorage 存 UUID
GET /me → 查 sessions 表（验证存在性+过期时间）→ JOIN users 返回
POST /logout → DELETE session 记录
```

```
    │   └── ttyd-linux      # Linux（手动下载放入）
    └── platform-tools/
    │   ├── adb.exe         # Windows（已有）
    │   └── ...             # Windows 文件
    └── platform-tools-linux/
        ├── adb             # Linux（手动下载放入）
        └── ...
```

### 4.2 Dockerfile（多阶段构建）

```dockerfile
# ---- Stage 1: 构建 ----
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run web:build

# ---- Stage 2: 运行 ----
FROM node:20-alpine
RUN apk add --no-cache libc6-compat  # node-pty 兼容
WORKDIR /app

# 复制构建产物
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/package*.json ./
RUN npm ci --omit=dev

# 复制 Linux 二进制
COPY bin/ttyd/ttyd-linux /usr/local/bin/ttyd
COPY bin/platform-tools-linux/ /usr/local/bin/platform-tools/
RUN chmod +x /usr/local/bin/ttyd /usr/local/bin/platform-tools/adb

# 数据库迁移脚本
COPY --from=builder /app/server/db/migrations ./dist/server/db/migrations

EXPOSE 3000
CMD ["node", "dist/server/index.js"]
```

### 4.3 docker-compose.yml

```yaml
version: '3.8'

services:
  # PostgreSQL 数据库
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: tscameratools
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: ${DB_PASSWORD:-postgres}
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 10s
      timeout: 5s
      retries: 5

  # Node.js 应用
  app:
    build: .
    depends_on:
      postgres:
        condition: service_healthy
    environment:
      - NODE_ENV=production
      - PORT=3000
      - DATABASE_URL=postgres://postgres:${DB_PASSWORD:-postgres}@postgres:5432/tscameratools
      - FRONTEND_URL=http://localhost
      - FEISHU_APP_ID=${FEISHU_APP_ID}
      - FEISHU_APP_SECRET=${FEISHU_APP_SECRET}
      - FEISHU_REDIRECT_URI=${FEISHU_REDIRECT_URI:-http://localhost:3000/auth/feishu/callback}
    volumes:
      - ./logs:/app/logs
    restart: unless-stopped

  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    depends_on:
      - app
    ports:
      - "80:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro
      - ./dist:/usr/share/nginx/html:ro
    restart: unless-stopped

volumes:
  pgdata:
```

### 4.4 nginx/nginx.conf

```nginx
server {
    listen 80;
    server_name _;

    # 前端静态文件
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://app:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Auth 路由代理
    location /auth/ {
        proxy_pass http://app:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # WebSocket - 终端
    location /terminal {
        proxy_pass http://app:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }

    # WebSocket - 内存监控
    location /memory-ws {
        proxy_pass http://app:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }

    # Socket.io
    location /socket.io/ {
        proxy_pass http://app:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }

    # 健康检查
    location /health {
        proxy_pass http://app:3000;
    }
}
```

## 五、后端静态文件托管

在 `server/index.ts` 中添加生产模式静态文件服务：

```typescript
// 生产模式：Nginx 未启用时的 fallback
if (process.env.NODE_ENV === 'production' && !process.env.NGINX_ENABLED) {
  const staticPath = path.join(__dirname, '..', 'dist')
  app.use(express.static(staticPath))
  app.get('*', (_req, res) => {
    res.sendFile(path.join(staticPath, 'index.html'))
  })
}
```

Docker 部署时 Nginx 直接托管 `dist/`，此代码作为备用。

### 7.3 认证流程验证
1. 访问 `http://localhost/login` → 点击飞书登录
2. OAuth 回调后检查 localStorage 中 `auth_token` 格式（应为 UUID，非 Base64）
3. 刷新页面验证 session 恢复（`GET /me` 返回用户信息）
4. 点击登出验证 session 删除（再次访问 `/me` 返回 401）
5. 检查 `device_history` 表是否有设备连接记录

## 九、飞书集团用户数据隔离说明

数据库设计支持**按租户隔离**：

- `users.tenant_key` 字段存储飞书租户标识
- 所有查询可通过 `WHERE tenant_key = ?` 过滤当前租户数据
- 后续如需多租户管理，只需在 API 层添加 tenant_key 过滤条件

查询示例：
```sql
-- 获取某租户下所有用户
SELECT * FROM users WHERE tenant_key = '${tenantKey}';

-- 获取某用户的设备历史
SELECT dh.* FROM device_history dh
JOIN users u ON dh.user_id = u.id
WHERE u.feishu_id = '${feishuId}';
```
