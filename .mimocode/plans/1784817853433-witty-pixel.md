# 本地 Agent 方案 — ADB + 终端本地化

## 问题

Web 部署在服务器时，所有 ADB/终端操作都在服务器上执行，无法触达用户本地电脑。
用户永远不应有权限操控服务器终端（安全风险）。

## 方案

创建轻量本地 WebSocket Agent，用户在本机运行后，Web 前端自动切换为本地模式。

```
用户浏览器 ──HTTP──→ 服务器 (122.51.90.193)   ← 认证、页面
用户浏览器 ──WS──→ 本地 Agent (localhost:9191) ← ADB、终端
```

---

## 实施步骤

### 1. 创建 `scripts/local-agent.ts`（本地 WebSocket 代理）

- WebSocket 服务器监听 `localhost:9191`
- 复用 `node-pty`（已在依赖中）+ `child_process`（Node 内置）
- 处理两种请求类型：

**ADB 操作**（JSON 消息，请求-响应模式）：
```
→ { type: "adb:check" }
← { type: "adb:result", id: "adb:check", available: true, version: "..." }

→ { type: "adb:devices" }
← { type: "adb:result", id: "adb:devices", devices: [{serial, model, status}] }

→ { type: "adb:root", serial: "..." }
← { type: "adb:result", id: "adb:root", success: true, message: "..." }

→ { type: "adb:remount", serial: "..." }
← { type: "adb:result", id: "adb:remount", success: true, message: "..." }
```

**终端操作**（复用现有协议，与 `server/services/terminal.ts` 一致）：
```
→ { type: "terminal:start", terminalType: "adb"|"local", serial?: "..." }
← { type: "ready", pid: ... }

→ { type: "input", data: "..." }
← { type: "output", data: "..." }

→ { type: "resize", cols: 80, rows: 30 }
→ { type: "kill" }
← { type: "exit", exitCode: ... }
```

- ADB 路径：优先 `process.env.ADB_PATH`，其次 `adb`（PATH 中查找）
- 终端 shell：Windows 用 `cmd.exe`，其他用 `bash`
- 日志：输出到 `logs/local-agent.log`

### 2. 创建 `src/hooks/useLocalAgent.ts`

- 尝试连接 `ws://localhost:9191`，维护连接状态
- 提供 `sendAdbCommand(type, params?)` Promise 方法
- 提供 `startTerminal(type, serial?)` → 返回 WebSocket 连接
- 自动重连，状态暴露：`{ available, connecting, error }`
- 不可用时返回 null，调用方优雅降级

### 3. 修改 `src/pages/DevicesWeb.tsx`

- 使用 `useLocalAgent()` hook
- Agent 可用时：通过 WebSocket 执行 ADB check/devices/root/remount
- Agent 不可用时：显示"请运行本地 Agent"引导 + 下载链接（保留之前的修改）
- 移除对 `fetchWithAuth('/api/adb/...')` 的依赖

### 4. 修改 `src/pages/LocalTerminal.tsx` + `XtermTerminal.tsx`

- `LocalTerminal.tsx`：通过 `useLocalAgent().startTerminal('local')` 获取 WebSocket
- `XtermTerminal.tsx`：新增 `wsOverride?: WebSocket` prop，允许外部传入已建立的 WebSocket（本地模式），而非内部创建连接
- 移除服务器端本地终端访问

### 5. 移除服务器本地终端（安全）

- `server/services/terminal.ts`：拒绝 `type=local` 的请求（返回错误）
- 仅保留 `type=adb`（服务器连接自身 USB 设备时使用）

### 6. 更新 `package.json`

- 添加 `"agent": "tsx scripts/local-agent.ts"` 脚本
- 更新启动说明

## 验证

1. **本地 Agent 启动**：`npm run agent` → 日志显示 `listening on ws://localhost:9191`
2. **ADB 检测**：打开浏览器访问服务器 → 设备页自动检测到本地 Agent 并显示设备列表
3. **ADB Shell**：点击 Connect → 终端连接本地 `adb shell`
4. **本地终端**：访问 `/terminal` → 打开用户本机 cmd.exe，非服务器 bash
5. **降级**：关闭本地 Agent → 设备页显示"请运行本地 Agent"引导
6. **安全性**：服务器 `/terminal?type=local` 返回错误

### 用户使用流程

```
1. git pull 最新代码
2. npm install
3. npm run agent          ← 启动本地代理
4. 浏览器打开 http://122.51.90.193
5. 设备页自动显示本地 ADB 设备，终端连接本地 CMD
```
